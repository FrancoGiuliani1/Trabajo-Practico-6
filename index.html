      • ¿Cómo es el manejo de las variables, tipos de datos y expresiones regulares en JS?.
    • Manejo de variables:
      • Hay tres tipos de variables estas son var, let, const.
      • var: Nos permite crear variables que pueden ser modificadas.
      • let: Esta variables nos permiten modificarlas, pero no se pueden re-declarar. Tambien puede servirnos para escribir un codigo mas limpio al usar funcines internas.
      • const: Se utiliza para crear variables de solo lectura, quiere decir que una vez que se crea esta no puede ser modificada.
    • Tipos de datos:
      • Undefined:Es cuando a una variable no se le asigna un valor como tal, estas son de tipo Undefined.
      • Boolean: Este puede presentar dos valores como tal. Es decir, comprueban que una condicion sea "esto o esto". Por ejemplo: verdaro o falso
      • Number: Este tipoo de dato es numerico y consta de 64 bits. Tiene tres valores: +Infinity, -Infinity y NaN ("Not a Number" o No es un número).
      • String: Se utiliza para crear cadenas, el primer elemento tiene la posición 0 y el segundo la posición 1.
      • Bigint: Se utiliza para almacenar y operar grandes numeros de una forma segura.
      • Symbol: Este es un valor unico y no se puede cambiar o sobreescribir y se puede utilizar como la clave una propiedad de objeto.
      • Object:Se utilizan para almacenar varios datos y colecciones de entidades más complejas asociadas con nombres clave. En JavaScript, los objetos impregnan casi todos los aspectos del lenguaje.
      ¿Cómo son todas las formas de declaración de funciones en JS?.
    • Las diferentes formas de declarar funciones son:
      • Function declaration: Este tipo de función se creará utilizando la palabra function, seguida de un nombre que identifique nuestra función, una lista de variables dentro de paréntesis y una llave {}. Las cuales determinaran el fin de la funcion.
      • Function expression: La sintaxis para crear estas funciones es similar a Function declaration. La diferencia entre estas es su definición de nuestra función no comienza por instrucción function y el nombre de la función es opcional. Este tipo de funciones las podemos almacenar en una variable.
      • IIFE (Immediately Invoked Function Expression): Este tipo de funciones se ejecutan inmediatamente y no son accesibles posteriormente. Es decir, solo se pueden utilizar una sola vez.
      • Shorthand method definition:
      • Arrow function: Esta función puede ser utilizada como método en la declaración de un objeto o en las clases de ES6.
      • Generator function: Esta funcion nos permite acortar la funcion a una sola linea y que el codigo sea mucho mas "limpio", ya que dejamos de lado el return.
      • Function constructor: Esta es una función especial que crea e inicializa una instancia de objeto de una clase. En JavaScript, se llama a un constructor cuando se crea un objeto usando la nueva palabra clave 'this'. El propósito es crear un nuevo objeto y establecer valores para cualquier propiedad de objeto existente.
      ¿Cuáles son los operadores y conversión entre tipos?.
      • Operadores de asignación: Este operador se utiliza para guardar un valor específico en una variable. El símbolo utilizado es '='.
      • Operadores lógicos: Los operadores lógicos son esenciales para la ejecución de aplicaciones complejas porque se utilizan para determinar qué instrucciones debe ejecutar el programa según ciertas condiciones. El resultado de cualquier operación que utilice operadores lógicos es siempre un valor lógico o booleano.
      • Operadores relacionales: Los operadores relacionales definidos en JavaScript son los mismos que los definidos en matemáticas: mayor que (>), menor que (<), mayor o igual que (>=), menor o igual que (<=), igual a ( ==), y (!=).
      • Operadores de comparación: Los operadores de comparación comparan sus operandos y devuelven un valor logico dependiendo de si la comparación es verdadera o falsa.
      • Operadores aritméticos: Los operadores aritméticos toman un valor numérico (literal o variable) como operando y devuelven un único valor numérico.
      • Operadores bit a bit: Los operadores bit a bit tratan sus operandos como un conjunto de 32 bits (ceros y unos), no como números decimales, hexadecimales u octales.
      • Operadores de cadena: En valores de cadena, el operador de concatenación (+) concatena dos valores de cadena y devuelve otra cadena que es la unión de los dos operandos.
      • Operador condicional (ternario): El operador condicional es el único operador de JavaScript que toma tres operandos. El operador puede tener uno de dos valores según una condición.
      • Operador coma: El operador coma (,) simplemente evalúa ambos operandos y devuelve el valor del último operando.
      • Operadores unarios: Una operación unaria es una operación con un solo operando.
      ¿Cuáles son las estructura de control de JS?.
  • Las estructuras de control son:
    • If - else: Las estructuras condicionales en javascript nos sirven para tomar decisiones en función de una condición que nosotros establecemos.
    • Switch: La estructura de switch permite probar la igualdad de una variable con una lista de valores. Cada valor se denomina case, y la variable que se activa se comprueba para cada case de switch.
    • While: Mientras se cumpla la condición realizar estas acciones. Cuando la condición deje de cumplirse salimos del bucle y continúa el flujo del programa.
    • Do while: Este bucle se realiza hasta que el usuario elija la opción de salir de el, mientras tanto este seguirá realizándose.
    • For: Cuando sabemos exactamente el número de veces que se repetirá el bucle; no dependen de una condición. Para estos casos existe el bucle 'for'.
    • Try catch: Estas estructuras se utilizan para ejecutar código de forma segura, ya que en el caso de que existan errores, podremos capturarlas y seguir ejecutando normalmente el código.
      • ¿Qué es el objeto dom en JS? Y ¿Cómo trabaja?.
  • ¿Qué es el objeto dom en JS?
    • Dom: La estructura del documento HTML está conformada por muchas etiquetas html. Estas están anidadas una dentro de la otra, formando un árbol de etiquetas relacionadas llamado 'dom'.
    • ¿Como Trabaja?
      • Al conectar js con html nos permite automatizar el documento html, como por ejemplo pulsar un boton, poner un contador, entre otras.
      • ¿Qué función tiene en JS el Object Map?.
  • El objeto Map contiene pares key-value y recuerda el orden de inserción original de las keys. Cualquier valor (tanto objetos como valores primitivos) se puede utilizar como key o como value.
      • ¿Cuáles son todos los eventos que trabaja JS?.
    • abort (onabort): Ocurre cuando el usuario detiene la carga de la imagen, deteniendo la carga de la página o realizando una acción que detiene la carga de la misma, como por ejemplo: salir de la página.
    • blur (onblur): El evento de onblur se activa cuando un elemento pierde el foco en la aplicación. El foco de la aplicación es la posición del cursor, por ejemplo, puede estar en un campo de texto, página, botón o cualquier otro elemento.
    • change (onchange): Se desata este evento cuando cambia el estado de un elemento de formulario, en ocasiones no se produce hasta que el usuario retira el foco de la aplicación del elemento.
    • click (onclick): Ocurre cuando se presiona un botón del mouse o se hace clic en un elemento de la página, generalmente un botón o un enlace.
    • dragdrop (ondragdrop): Se produce cuando un usuario suelta algo que había arrastrado sobre la página web.
    • error (onerror): Se produce cuando no se puede cargar un documento o una imagen y esta queda rota.
    • focus (onfocus): El evento onfocus es lo opuesto al onblur. Ocurre cuando un elemento en una página o ventana tiene el foco en la aplicación.
    • keydown (onkeydown): Este evento ocurre cuando el usuario presiona una tecla, la suelte o no. Esto sucede en el momento de la pulsación.
    • keyup (onkeyup): Un evento de onkeypress ocurre cuando el usuario mantiene presionada una tecla durante un cierto período de tiempo. . Antes de este evento se produce un onkeydown en el momento que se pulsa la tecla..
    • load (onload): Este evento se activa cuando una página o imagen de JSha terminado de cargarse.
    • mousedown (onmousedown): El evento onmousedown se activa cuando el usuario hace clic en un elemento de la página. onmousedown ocurre cuando se presiona el botón, ya sea que se suelte o no.
    • mousemove (onmousemove): Se produce cuando el cursor se mueve por la página.
    • mouseout (onmouseout): El evento onmouseout se activa cuando el cursor del mouse abandona el área ocupada del elemento de la página.
    • mouseover (onmouseover): Este evento se activa cuando el cursor del mouse ingresa al área ocupada por el elemento de la página.
    • mouseup (onmouseup): Este evento ocurre cuando el usuario suelta el botón del mouse previamente presionado.
    • move (onmove): Este evento se activa cuando se mueve una ventana del navegador o un frame.
    • resize (onresize): Un evento que ocurre cuando cambia la ventana o el marco del navegador (si la página tiene uno).
    • reset (onreset): Este evento se adjunta al formulario y se activa cuando el usuario hace clic en el botón de reinicio o reset del formulario.
    • select (onselect): Se ejecuta cuando un usuario realiza una selección de un elemento de un formulario.
    • submit (onsubmit): Esto sucede cuando el visitante hace clic en el botón para enviar el formulario. Se ejecuta antes de enviarse.
    • unload (onunload): El evento de descarga se ejecuta cuando abandona la página, ya sea porque se hizo clic en un enlace que nos llevó a otra página o porque se cerró la ventana del navegador.
    • scroll (onscroll): Este evento se produce cuando se realiza scroll o desplazamiento. Este desplazamiento se puede realizar en toda la página o en los elementos de la página que tienen su propia área de scroll.
    • DblClick (ondblclick): Este evento se activa cuando el usuario hace doble clic en un elemento de formulario o link.
